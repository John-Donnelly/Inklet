Fix session persistence bugs + add SessionTests

PersistSession now calls SaveCurrentTabState() first so the active tab cursor
position is always captured, not just on app close and tab switch.

Last-tab reset in CloseTabAsync now calls PersistSession() after clearing state
so a crash between reset and app close restores an empty tab, not the old one.
CursorPosition is also explicitly zeroed on reset.

TabSession and PersistedTabData promoted from internal to public (consistent
with DocumentState) to allow direct testing without InternalsVisibleTo.

SessionTests.cs adds 18 tests covering all four session states: new empty tab,
untitled with content, opened file unchanged, opened file with unsaved changes.
Also covers TabTitle, PersistedTabData projection, restore logic, cursor and encoding round-trips.
